## OIP Types

- **Standards Track** — Changes to the product, APIs, infrastructure, or frontend
- **Informational** — Design guidelines, best practices, or general information
- **Process** — Changes to development processes, tooling, or governance

## Status Lifecycle

```
Draft → Review → Accepted → Implemented
         ↓
    Deferred (parked for later)

Draft/Review → Rejected | Withdrawn
Implemented → Superseded (by a later OIP)
```

## How to Submit an OIP

1. Copy [TEMPLATE.md](TEMPLATE.md) to `oip-NNNN-short-title.md`
2. Fill in all required sections (Abstract should be understandable in 30 seconds)
3. Add your OIP to the status table in [README.md](README.md)
4. Submit for review via PR or direct discussion

## Required Sections

Every OIP must include:
- **Preamble** — Metadata header block
- **Abstract** — 2-3 sentence summary
- **Motivation** — Why this change is needed
- **Specification** — Concrete technical details
- **Rationale** — Why this approach over alternatives
- **Backwards Compatibility** — Impact on existing behavior
- **Security Considerations** — Authentication, rate limiting, abuse vectors
- **Open Issues** — Unresolved questions for discussion
