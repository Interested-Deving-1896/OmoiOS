## Brand Positioning

### What OmoiOS Is

OmoiOS is **autonomous engineering execution infrastructure**—a spec-driven orchestration platform that turns feature descriptions into tested, shipped code without requiring engineers to babysit every step.

### What OmoiOS Is Not

- Not an IDE plugin (that's Cursor, Kiro)
- Not a chat interface (that's Claude Code, ChatGPT)
- Not a single-task coding agent (that's Codex)
- Not a no-code tool (users are technical, write specs, review PRs)

### Brand Pillars

| Pillar | Expression |
|--------|------------|
| **Autonomous** | "You sleep. Agents ship." |
| **Spec-Driven** | "Specs are contracts. Agents execute against them." |
| **Transparent** | "Open source. Self-hostable. Full traceability." |
| **Reliable** | "Self-healing execution. Validation loops. Guardian oversight." |
| **Scalable** | "Add agents, not headcount. Parallel execution without chaos." |

## Target Audience

### Primary Persona: Engineering Manager "Alex"

**Role**: Engineering Manager at 20-person SaaS company
**Pain**: Roadmap grows faster than team capacity. Hiring takes 3+ months.
**Goal**: Ship more features without burning out existing team or waiting for new hires.
**Behavior**: Reviews PRs, writes specs, attends standups. Doesn't want to micromanage.
**Quote**: *"I need capacity tonight, not in three months."*

**How OmoiOS Helps**:
- Alex writes a spec for the feature
- OmoiOS generates requirements, design, and task breakdown
- Agents execute overnight while Alex sleeps
- Morning: tested PR ready for Alex's review
- Alex approves, merges, ships

---

### Secondary Persona: Senior IC Engineer "Jordan"

**Role**: Senior Engineer at mid-size company
**Pain**: Repetitive implementation work drains energy from architecture and innovation.
**Goal**: Focus on high-leverage design work; offload boilerplate to agents.
**Behavior**: Uses Claude Code for quick tasks, but wants something for larger features.
**Quote**: *"I want to design the system, not write the CRUD endpoints."*

**How OmoiOS Helps**:
- Jordan specs out the API design
- OmoiOS agents implement the endpoints, tests, and documentation
- Jordan reviews the PR, focuses on architecture decisions
- Jordan moves on to the next high-value design task

---

### Tertiary Persona: Solo Founder "Morgan"

**Role**: Solo technical founder building MVP
**Pain**: Limited time, wearing all hats, need to move fast.
**Goal**: Ship features quickly without sacrificing code quality.
**Behavior**: Active on HN, Indie Hackers, Twitter. Cost-conscious but values time.
**Quote**: *"I can't afford to hire, but I can write specs all day."*

**How OmoiOS Helps**:
- Morgan describes features in plain English
- OmoiOS plans and executes while Morgan handles sales, marketing, support
- Morgan wakes up to completed PRs
- Morgan's velocity increases without burning out

> "OmoiOS is autonomous engineering execution. You describe a feature, approve a plan, and agents deliver a tested PR overnight. Not an IDE plugin—cloud infrastructure that runs while you sleep. Open source, self-hostable, and spec-driven instead of prompt-driven."

### One-Liner (10 words)

> "Start a feature before bed. Wake up to a PR."

For **engineering teams who need to ship faster than they can hire**, OmoiOS is an **autonomous execution platform** that **turns specs into shipped code overnight**—unlike **Kiro, Codex, or Cursor** which require you to be at your desk, OmoiOS **runs the full pipeline autonomously in the cloud** with **full traceability and self-healing execution**.

## Messaging by Use Case

### Use Case 1: Backlog Clearance

**Headline**: "Clear your backlog without clearing your calendar."

**Body**: Your roadmap grows daily. Recruiting takes months. OmoiOS adds agent capacity tonight—describe what you want, approve the plan, and wake up to shipped work. No headcount required.

**CTA**: "Start Free"

---

### Use Case 2: Overnight Execution

**Headline**: "You sleep. Agents ship."

**Body**: Write a spec, close your laptop. Agents execute in isolated sandboxes with dependency-aware sequencing. Tests fail? Agents debug and retry. Validation passes? PR created. You review in the morning.

**CTA**: "See How It Works"

---

### Use Case 3: Spec-Driven Quality

**Headline**: "Garbage spec, garbage out. Great spec, great PR."

**Body**: OmoiOS enforces a spec pipeline—Explore → Requirements → Design → Tasks—before any code is written. By the time agents start coding, they have machine-checkable acceptance criteria. No drift. No surprises.

**CTA**: "Learn About Specs"

---

### Use Case 4: BYO API Keys (Power Users)

**Headline**: "Unlimited workflows. Your keys. Your cost."

**Body**: Already paying for Anthropic or OpenAI? Bring your own API keys and run unlimited workflows on OmoiOS. We handle the orchestration; you control the LLM costs. $19/month for platform access.

**CTA**: "Connect Your Keys"

## Competitive Messaging

### vs Kiro (AWS)

**Their Position**: Spec-driven IDE for interactive coding

**Our Differentiation**: "Kiro helps you code at your desk. OmoiOS codes for you in the cloud. Both are spec-driven, but OmoiOS runs the full pipeline autonomously while you're away."

**When to Choose OmoiOS**: You want to approve a plan and walk away, not code interactively.

---

### vs OpenAI Codex

**Their Position**: Cloud coding agent for individual tasks

**Our Differentiation**: "Codex is prompt-driven—assign individual tasks. OmoiOS is spec-driven—describe a full feature, get a complete PR. Also open-source, self-hostable, and multi-model."

**When to Choose OmoiOS**: You want full feature lifecycle, not individual task execution.

---

### vs Claude Code

**Their Position**: Terminal-based pair programming

**Our Differentiation**: "Claude Code is your pair programmer—excellent for interactive development. OmoiOS is your overnight engineering team—write a spec, agents execute while you sleep."

**When to Choose OmoiOS**: You want autonomous execution, not pair programming.

---

### vs Cursor

**Their Position**: AI-powered code editor

**Our Differentiation**: "Cursor makes you faster at coding. OmoiOS eliminates the coding—you write specs, agents deliver PRs. Different tool for a different workflow."

**When to Choose OmoiOS**: You want to write specs and review PRs, not write code.

## Visual Identity

### Design Direction

**Style**: Linear/Arc aesthetic—clean, minimal, white-space-heavy, professional SaaS

**Key Elements**:
- Light theme default with smooth animations
- Gentle shadows and subtle gradients for depth
- Structured block-based content (Notion-style)
- Collapsible sidebar navigation (Obsidian-style)
- React Flow dependency graphs for visual impact

### Color Palette

| Role | Color | Usage |
|------|-------|-------|
| Primary | Deep blue | CTAs, links, brand moments |
| Secondary | Slate gray | Body text, secondary elements |
| Accent | Emerald green | Success states, completion |
| Warning | Amber | Alerts, intervention needed |
| Error | Rose | Failures, blocking issues |

### Typography

- **Headlines**: Inter, bold, tight tracking
- **Body**: Inter, regular, comfortable line height
- **Code**: JetBrains Mono, for technical content
