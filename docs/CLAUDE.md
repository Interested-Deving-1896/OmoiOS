# OmoiOS Documentation

This is the root documentation directory for OmoiOS — a spec-driven, multi-agent orchestration system that turns feature ideas into PRs using parallel agent swarms.

## Documentation Structure

| Directory | Purpose | Key Files |
|-----------|---------|-----------|
| `architecture/` | System architecture deep-dives | 01-planning-system.md through 16-service-catalog.md |
| `design/` | Design documents by domain | sandbox-agents/, workflows/, auth/, frontend/, etc. |
| `figma_prompts/` | Figma Make prompts for UI | 11 sequential prompts for building the complete UI |
| `implementation/` | Implementation logs and guides | services/, tracking/ |
| `page_flows/` | Page-by-page UI flow docs | 24 documented flows |
| `proposals/` | OmoiOS Improvement Proposals (OIPs) | OIP-0001 through OIP-0007, TEMPLATE.md |
| `requirements/` | Requirements by system | agents/, monitoring/, billing/ |
| `research/` | Research reports and findings | Various technology evaluations |
| `user_journey/` | End-to-end user journey docs | 16 files from onboarding to billing |
| `assets/` | Static assets (images, GIFs, banners) | demo GIFs, banner.svg, social-preview.png |
