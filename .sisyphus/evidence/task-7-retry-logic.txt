Retry logic verification:

Function: fetch_with_retry()
  - 3 attempts maximum
  - Exponential backoff: 1s, 2s, 4s
  - Uses curl -fsS with --max-time 10
  - Returns response body on stdout, or 1 on failure
  - Guarded against set -e via "if resp=$(...)" pattern

Backoff sequence:
  Attempt 1: delay=1s, fail -> sleep 1s, delay becomes 2s
  Attempt 2: delay=2s, fail -> sleep 2s, delay becomes 4s
  Attempt 3: delay=4s, fail -> return 1 (max attempts reached)

Fail-closed behavior:
  - If any alias fetch returns non-200 after retries: exit 1
  - If unknown broker kind: exit 1
  - If auth.json validation fails (type not "api" or "oauth"): exit 1
  - All failures happen BEFORE VNC/OpenCode startup
